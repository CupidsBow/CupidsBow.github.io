CF1286E Fedya the Potter Strikes Back

发表于 2022-08-05 更新于 2024-07-21 分类于 杂题 本文字数： 2.8k 阅读时长 ≈ 3 分钟

kmp 好题，着重对 fail 数组的理解

题意

给定字符串 sss 和序列 www ，初始均为空。
总共 nnn 次操作，每次在 sss 末端添加一个小写字符 ccc ，在 www 末端添加一个整数 www ，每次添加后求出所有满足 s[1,r−l+1]=s[l,r]s[1,r-l+1] = s[l,r]s[1,r−l+1]=s[l,r] 的区间 [l,r][l,r][l,r] 的最小值之和。
强制在线。
其中 1≤n≤6⋅105,0≤w≤230−1,c∈[a,z]1 \leq n \leq 6 \cdot 10^5, 0 \leq w \leq 2^{30}-1, c \in [a, z]1≤n≤6⋅105,0≤w≤230−1,c∈[a,z] 。

思路

考虑到条件 s[1,r−l+1]=s[l,r]s[1,r-l+1] = s[l,r]s[1,r−l+1]=s[l,r] 不难想到每次添加字符后的答案即为上一次的答案加上当前串的所有 borderborderborder 的贡献。
那么需要每次添加字符时实时维护当前串的 borderborderborder 集合 TTT 与它们的总贡献 sumsumsum 。

  • 对于 TTT ，考虑添加一个字符后，对于 TTT 中的每一个 borderborderborder ，若它的后一个字符不是 ccc ，则需要将它从 TTT 中删除，而如果有 c=s[1]c = s[1]c=s[1] 则需要将 111 也加入集合。对于每次添加操作，最多会往集合中加入一个数，每个数最多被删去一次，所以维护 TTT 的复杂度是均摊 O(n)O(n)O(n) 的。
    对于删去不满足条件的 borderborderborder 的操作，可以考虑在维护 failfailfail 数组的同时维护一个 preprepre 数组，其中 preipre_iprei​ 表示前 i−1i - 1i−1 个字符组成的前缀的 borderborderborder 集合中最长的不满足条件的 borderborderborder 长度，每次暴力遍历 preprepre 即可，由于均摊操作复杂度有保证。
  • 对于 sumsumsum ，考虑删去 TTT 中的一个 borderborderborder 后需要减掉其对应的贡献，这个可以用一个单调栈 + 二分解决，对于剩下的 borderborderborder 均往后拓展了一位，需要将其贡献与 www 取 minminmin ，若新添加 111 则需要加上 www 的贡献。
    对于将贡献与 www 取 minminmin 的操作，可以考虑对每种贡献 w′w'w′ 用 mapmapmap 记录一个数量，每次暴力将 mapmapmap 中大于 www 的数更新为 www 。由于每种数字只会被更新一次（更新完即合并了），所以加上 mapmapmap 的复杂度后维护 sumsumsum 的复杂度是均摊 O(n⋅logn)O(n \cdot logn)O(n⋅logn) 的。
    注意结果可能会爆 long longlong\ longlong long ，用 int128int128int128 存答案即可，维护的总复杂度为 O(n⋅logn)O(n \cdot logn)O(n⋅logn) 。
