C. Mr. Credo

tag

  • 计算几何
  • 差分

题意

给定 nnn 个光源的坐标 x,yx, yx,y 与照射角度范围，保证所有角度加起来小于 2⋅π2 \cdot \pi2⋅π ，找一个半径为 111 的圆使得不被照到。
其中 1≤n≤2⋅105,−50≤xi,yi≤501 \leq n \leq 2 \cdot 10^5, -50 \leq x_i, y_i \leq 501≤n≤2⋅105,−50≤xi​,yi​≤50 。

思路

构造的圆心坐标范围可以达到 [−109,109][-10^9, 10^9][−109,109] ，光源的坐标范围为 [−50,50][-50, 50][−50,50] 相差很大，可以认为所有光源均在原点，找一个没有被照射的角度将圆取在尽可能远的位置即可。
复杂度 O(n)O(n)O(n) 。

D. Deep Purple

tag

  • 后缀自动机
  • 树链剖分
  • 线段树

题意

给定长为 nnn 的小写字母串 sss ， qqq 次询问，每次询问区间 [l,r][l, r][l,r] 的最长 borderborderborder 。
其中 1≤n,q≤2⋅105,1≤li,ri≤n1 \leq n, q \leq 2 \cdot 10^5, 1 \leq l_i, r_i \leq n1≤n,q≤2⋅105,1≤li​,ri​≤n 。

思路

考虑单次询问，即求最大的 iii 满足 lcs(i,r)>i−llcs(i, r) > i - llcs(i,r)>i−l ，对 sss 建出 SAMSAMSAM ，记 posxpos_xposx​ 为 endpos=xendpos = xendpos=x 的节点在后缀树上的编号， iii 的条件即为存在节点 xxx 满足：

  • xxx 为 posrpos_rposr​ 的祖先节点
  • xxx 的子树中存在节点 iii 使得 len(x)+l>endposilen(x) + l > endpos_ilen(x)+l>endposi​

求出满足条件的最大的 iii ，若 i≥li \geq li≥l ，询问答案即为 i−l+1i - l + 1i−l+1 。
对于单次询问，可以先将后缀树上 posrpos_rposr​ 到根的路径上每个节点加上权值 len(x)+llen(x) + llen(x)+l ，再从大到小枚举 iii ，找到第一个 iii 满足： iii 到根的路径上存在节点的标记值大于 iii 。用 iii 更新答案即可。
对于多次询问，可以将询问离线下来处理，从大到小枚举 iii ，遇到询问右端点 rrr 就将 posrpos_rposr​ 到根这段路径打上权值 len(x)+llen(x) + llen(x)+l ，再查询 iii 能更新的询问，更新完成或者不能再被更新后就把对应询问打上的标记删去。感觉这部分将询问一起处理有点类似整体二分的过程。
现在需要能够加 / 删标记与查询标记最大值，可以树剖 + 线段树维护。
对于 posrpos_rposr​ 到根的这段路径，将它拆成 logloglog 条重链，分两颗线段树 tr0,tr1tr0, tr1tr0,tr1 来维护， tr0tr0tr0 维护加上的 lll 的最大值， tr1tr1tr1 维护 len(x)+llen(x) + llen(x)+l 的最大值，加 / 删标记时对于每条重链只将 lll 加 / 删在链尾并在线段树上修改对应节点的值（树上每个节点维护一个 setsetset 记录加 / 删在当前节点的权值，线段树只负责维护一个区间 maxmaxmax ）。
查询时对每条重链分为两部分，一部分为 tr1tr1tr1 直接查询的区间的值，另一部分为最深的节点的 lenlenlen 加上 tr0tr0tr0 查询的整条链的值，因为 len(x)len(x)len(x) 随着 xxx 的深度增加而单调增，所以查询结果一定在其中一个中。
复杂度 O(n⋅log2n)O(n \cdot log^2n)O(n⋅log2n) 。

其中 2≤n,q≤105,0≤x≤1092 \leq n, q \leq 10^5, 0 \leq x \leq 10^92≤n,q≤105,0≤x≤109 。

G. Green Day

tag

  • 构造

题意

构造一张图，边总共有k种颜色，要求每种颜色的边均构成一棵生成树且任意两点在任意两棵生成树上的路径的交集只含有路径端点。
其中 1≤k≤1001 \leq k \leq 1001≤k≤100 。

思路

选定点数 n=2⋅kn = 2 \cdot kn=2⋅k ，结果为一个完全图。
对于 i∈[1,k]i \in [1, k]i∈[1,k] ，使 iii 向 i+j(i+1≤j≤i+k)i + j(i + 1 \leq j \leq i+k)i+j(i+1≤j≤i+k) 连边， i+ki + ki+k 向 i+k+j(1≤j≤n−1−k)i + k + j(1 \leq j \leq n-1-k)i+k+j(1≤j≤n−1−k) 连边即可。
复杂度 O(k)O(k)O(k) 。

H. Hans Zimmer

tag

  • 数学
  • 期望

题意

给定 w∗hw * hw∗h 的矩形，随机横着或者竖着切 nnn 次，求最后面积最小的矩形的期望面积。
其中 1≤w,h≤103,1≤n≤1061 \leq w, h \leq 10^3, 1 \leq n \leq 10^61≤w,h≤103,1≤n≤106 。

思路

结论：

  • 长度为 111 的线段切成 nnn 段，其中最小段的期望长度为 1n2\frac{1}{n^2}n21​

枚举横着切了 iii 刀，结果为：

  • ∑i=0n(in)2n⋅(i+1)2⋅(n−i+1)2\sum\limits_{i = 0}^{n} \frac{\binom{i}{n}}{2^n \cdot (i + 1)^2 \cdot (n - i + 1)^2}i=0∑n​2n⋅(i+1)2⋅(n−i+1)2(ni​)​

但交上去之后发现精度寄了， nnn 比较大的时候算 2n2^n2n 直接算裂了。
考虑取对数后再 expexpexp 回去即可。
复杂度 O(n)O(n)O(n) 。

I. Ivan Dorn

tag

  • 单调栈
  • 线段树

题意

给定长为 nnn 的序列 aaa ，定义一个区间 [l,r][l, r][l,r] 为 NBNBNB 区间当且仅当 al=ar,max[l,r]=ala_l = a_r, max[l, r] = a_lal​=ar​,max[l,r]=al​ 。
qqq 次询问，每次询问 [l,r][l, r][l,r] 内最长的 NBNBNB 区间的长度。
其中 1≤n,q≤5⋅105,1≤l,r≤n1 \leq n, q \leq 5 \cdot 10^5, 1 \leq l, r \leq n1≤n,q≤5⋅105,1≤l,r≤n 。

思路

对于一个询问 [l,r][l, r][l,r] ，找出 [l,r][l, r][l,r] 的区间 maxmaxmax 最左边和最右边的位置 posl,posrposl, posrposl,posr ，则 [l,r][l, r][l,r] 被 posl,posrposl, posrposl,posr 分割为 2/32 / 32/3 部分，显然中间部分是没有用的，剩下来只要求出左边部分最长能往右延申的长度 lenllenllenl 和右边部分最长能往左延申的长度 lenrlenrlenr ，再和 posr−poslposr - poslposr−posl 取 maxmaxmax 即可，由于左右部分被区间 maxmaxmax 分隔开，延伸长度一定不会跨过询问区间外，直接取 maxmaxmax 是正确的。
每个位置的延申长度可以单调栈预处理，区间 maxmaxmax 套线段树即可。
复杂度 O(n⋅logn)O(n \cdot logn)O(n⋅logn) 。

K. Korn

tag

  • 图论
  • DFS树

题意

给定 nnn 和点 mmm 条边的无向连通图，求 NBNBNB 点的个数。
一个点为 NBNBNB 点当且仅当从这个点开始走的所有回路均为原图的一条欧拉回路。
其中 3≤n≤2⋅105,n−1≤m≤5⋅1053 \leq n \leq 2 \cdot 10^5, n - 1 \leq m \leq 5 \cdot 10^53≤n≤2⋅105,n−1≤m≤5⋅105 。

思路

首先将存在奇数度数点的情况特判掉。
剩下来的情况中一个点为 NBNBNB 点当且仅当所有环均经过它，跑出原图的 dfsdfsdfs 树判一下即可。
复杂度 O(n)O(n)O(n) 。
