A. MEX-Query

tag

  • 线段树

题意

序列长为 nnn ， qqq 次询问求区间 mexmexmex 。
其中 1≤n,q≤1061 \leq n, q \leq 10^61≤n,q≤106 。

思路

询问离线下来按 rrr 排序，开权值树记每个数最后一次出现位置的最小值即可。
复杂度 O(n⋅logn)O(n \cdot logn)O(n⋅logn) 。

E. Another Short Problem

tag

  • 期望
  • 计算几何

题意

给定 nnn 个点，每个点有 ppp 的概率被选中，求选中的点构成的凸包的期望顶点数。
其中 1≤n≤501 \leq n \leq 501≤n≤50 。

思路

对于多面体有欧拉公式 F+V−E′=2F + V - E' = 2F+V−E′=2 ， FFF 表示面数， E′E'E′ 表示棱数， VVV 表示顶点数，在期望条件下也成立。
所以有 E(F)+E(V)−E(E′)=2E(F) + E(V) - E(E') = 2E(F)+E(V)−E(E′)=2 即 E(V)=2+E(E′)−E(F)E(V) = 2 + E(E') - E(F)E(V)=2+E(E′)−E(F) 。
期望面数可以通过枚举三个点表示的平面求得。由于不存在四点共面，所以所有面均为三角形，可以得到 E(E′)=3⋅E(F)2E(E') = \frac{3 \cdot E(F)}{2}E(E′)=23⋅E(F)​ ，所以 E(V)=2+E(F)2E(V) = 2 + \frac{E(F)}{2}E(V)=2+2E(F)​ 。
但上述公式是对于凸多面体而言，对于不能构成凸多面体的情况（如点数为 0,1,20, 1, 20,1,2 时）是没有被计算到的。
具体的：

  • 点数 V=0V = 0V=0 时，此时有 2+E′−F=02 + E' - F = 02+E′−F=0 ，而我们计算时按照 E′=F=0E' = F = 0E′=F=0 ， 2+E′−F=22 + E' - F = 22+E′−F=2 答案多算了 222 。
  • 点数 V=1V = 1V=1 时，此时有 2+E′−F=12 + E' - F = 12+E′−F=1 ，而我们计算时按照 E′=F=0E' = F = 0E′=F=0 ， 2+E′−F=22 + E' - F = 22+E′−F=2 答案多算了 111 。
  • 点数 V=2V = 2V=2 时，此时有 2+E′−F=22 + E' - F = 22+E′−F=2 ，而我们计算时按照 E′=F=0E' = F = 0E′=F=0 ， 2+E′−F=22 + E' - F = 22+E′−F=2 答案多算了 000 。

把多计算的减去就可以了。
复杂度 O(n4)O(n^4)O(n4) 。

J. Dividing Area

tag

  • 计算几何
  • 并查集

题意

给定二维平面 nnn 个点。
qqq 次操作，每次操作如下：

  • 选两个点连一条线段，保证所有线段除了端点外不会相交。
  • 询问有向线段 ababab 左侧平面的面积，无限输出 −1-1−1 。

其中 1≤n≤2⋅105,1≤q≤1061 \leq n \leq 2 \cdot 10^5, 1 \leq q \leq 10^61≤n≤2⋅105,1≤q≤106 。

思路

若将所有操作反过来，则加边的操作变为断边，即为合并两个相邻的平面。
考虑如何计算一个凸包的面积： area=∑i=0ncross(p[i],p[(i+1)%n])area = \sum\limits_{i = 0}^{n} cross(p[i], p[(i+1)\%n])area=i=0∑n​cross(p[i],p[(i+1)%n]) ，可以看成每条边的贡献之和。
类似的，对将要加入的线段 uvuvuv ，将其分为两条有向线段 uv,vuuv, vuuv,vu ，分别给其左右平面编号。则每条线段对其左侧平面的贡献可以叉积求得。
一个平面的总面积即为所有包围它的线段的贡献和。
所有询问线段可以分为三种情况：

  • 询问线段左侧为一个凸多边形，则线段的贡献和即为所求面积。

  • 询问线段左侧为一个 CCC 字，则线段的正反面贡献相互抵消，贡献和为 000 ，面积为无限。

  • 询问线段左侧为一个 CCC 字上附着若干个多边形， CCC 字部分的贡献相互抵消结果应为 000 ，另外减去多边形的面积，最后贡献和为负数，面积为无限。

则可以得到，平面的面积为无限当且仅当它的贡献和小于等于 000 。
则可以将给定询问倒过来处理，每次加边操作相当于合并两个相邻平面，并查集维护平面贡献即可。初始化将所有共端点的线段的平面合并即可。
复杂度 O(q⋅logq)O(q \cdot logq)O(q⋅logq) 。

K. Tree Problem

tag

  • 树链剖分
  • 线段树

题意

给定 nnn 个节点的树， qqq 次操作，每次操作如下：

  • 计算路径 ababab 上有多少条边被标记。
  • 将路径 ababab 上的边取消标记，并将直接连接在路径 ababab 上的所有边记上标记。

初始所有边均被标记。
其中 1≤n≤2⋅105,1≤q≤3⋅1051 \leq n \leq 2 \cdot 10^5, 1 \leq q \leq 3 \cdot 10^51≤n≤2⋅105,1≤q≤3⋅105 。

思路

固定 111 为树根，考虑两种映射边的方式，一种是将边映射到它连接的儿子上，即为方案 111 ，一种是映射到它连接的父节点上，即为方案 222 。
考虑树剖，剖完之后用两棵线段树分别维护方案 1,21, 21,2 ，树 111 维护所有重边，树 222 维护所有轻边。
对于修改链 ababab 的操作，直接在链上的边可以直接按方案 111 修改它的子节点，对于连接在路径上的边，这些边是所有链上节点连向的子节点的边并上 ababab 的 lcalcalca 连向父节点的边，前者可以按方案 222 修改对应节点，后者直接特判，在修改前者时会涉及 logloglog 条重边，对于这些边也按方案 111 修改即可。
对于查询操作，树剖之后相当于查询 logloglog 条重链和 logloglog 条轻边，对于重链直接按方案 111 查询，对于轻边，则需要比较方案 1,21, 21,2 的结果，这里再记录一个修改操作的时间戳即可。
复杂度 O(n⋅log2n)O(n \cdot log^2n)O(n⋅log2n) 。
