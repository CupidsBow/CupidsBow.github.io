B. Reverse Game

tag

  • 博弈

题意

长为 nnn 的 010101 串， AliceAliceAlice 和 BobBobBob 每次可以翻转一个等于 10,110,100,101010, 110, 100, 101010,110,100,1010 的子串，不能操作的人输，问谁赢。
其中 1≤n≤1061 \leq n \leq 10^61≤n≤106 。

思路

每次翻转会减少 111 或 222 个逆序对，统计一下总逆序对数即可。
打个表可以发现除了逆序对数等于 333 的倍数都是 AliceAliceAlice 赢。
复杂度 O(n)O(n)O(n) 。

E. Divisible by 3

tag

  • 数学

题意

长为 nnn 的序列 aaa ，求 nbnbnb 子串的个数。
数列 aaa 的一个子串 a[l,r]a[l, r]a[l,r] 是 nbnbnb 当且仅当它的权值是 333 的倍数。
一个数列 b[1,2,...,m]b[1, 2, ..., m]b[1,2,...,m] 的权值为 ∑i=1n∑j=i+1nbi⋅bj\sum\limits_{i = 1}^{n}\sum\limits_{j = i + 1}^{n} b_i \cdot b_ji=1∑n​j=i+1∑n​bi​⋅bj​ 。
其中 1≤n≤5⋅105,0≤ai≤1091 \leq n \leq 5 \cdot 10^5, 0 \leq a_i \leq 10^91≤n≤5⋅105,0≤ai​≤109 。

思路

将 aaa 每一项模 333 后为一个 012012012 数列。
000 没有影响，只要计算 1,21, 21,2 对总权值的贡献即可，记 c1,c2c_1, c_2c1​,c2​ 为子串中 1,21, 21,2 出现的次数，则子串总权值为 c1⋅(c1−1)2+4⋅c2⋅(c2−1)2+2⋅c1⋅c2\frac{c_1 \cdot (c_1 - 1)}{2} + 4 \cdot \frac{c_2 \cdot (c_2 - 1)}{2} + 2 \cdot c_1 \cdot c_22c1​⋅(c1​−1)​+4⋅2c2​⋅(c2​−1)​+2⋅c1​⋅c2​ 。
枚举右端点与左端点对应的 c1,c2c_1, c_2c1​,c2​ 值并计算即可。
复杂度 O(n⋅32)O(n \cdot 3^2)O(n⋅32) 。

tag

  • dp
  • 单调栈
  • 树状数组

题意

长为 nnn 的排列 aaa ，单次操作可以将 ai(l≤i≤r)a_i(l \leq i \leq r)ai​(l≤i≤r) 赋值成 min(al,al+1,...,ar)min(a_l, a_{l + 1}, ..., a_r)min(al​,al+1​,...,ar​) ，求能得到的不同的数列个数，模 109+710^9 + 7109+7 。
其中 1≤n≤30001 \leq n \leq 30001≤n≤3000 。

思路

双倍经验AGC058 B - Adjacent Chmax，只不过取 minminmin 变成取 maxmaxmax 。
考虑如下的 dpdpdp ：

  • 记 dp[i][j]dp[i][j]dp[i][j] 为考虑了前 iii 个数， ai=ja_i = jai​=j 的数列个数。
  • pospospos 为 iii 往前第一个小于 aia_iai​ 的位置，不存在则为 000 ，用单调栈维护。
  • 则有 dp[i][j]+=∑j=posi−1∑k=1ndp[j][k]dp[i][j] += \sum\limits_{j = pos}^{i - 1}\sum\limits_{k = 1}^{n} dp[j][k]dp[i][j]+=j=pos∑i−1​k=1∑n​dp[j][k] （用 aia_iai​ 往前扩展）。
  • dp[i][a[x]]+=∑j=1a[x]dp[i−1][j]dp[i][a[x]] += \sum\limits_{j = 1}^{a[x]} dp[i - 1][j]dp[i][a[x]]+=j=1∑a[x]​dp[i−1][j] （aia_iai​ 被前面比它小的数覆盖， xxx 为当前状态下单调栈内存的下标）。
  • 当更新 dp[i]dp[i]dp[i] 后，对 j∈[pos+1,i−1]j \in [pos + 1, i - 1]j∈[pos+1,i−1] 这一段的 dp[j][a[i]]dp[j][a[i]]dp[j][a[i]] 的值也要对应地更新，因为在加入 aia_iai​ 之前是不存在情况使得 aj=aia_j = a_iaj​=ai​ 的，加入 aia_iai​ 后 aia_iai​ 可以往前延伸，会多出这一种情况。

H. AND = OR

tag

  • 思维
  • 线段树

题意

长为 nnn 的数列 aaa ， qqq 次询问，每次询问能否将 (al,al+1,...,ar)(a_l, a_{l + 1}, ..., a_r)(al​,al+1​,...,ar​) 分为两个非空集合使得一个集合内元素的或等于另一个集合内元素的与。
其中 1≤n,q≤105,0≤ai≤230−11 \leq n, q \leq 10^5, 0 \leq a_i \leq 2^{30} - 11≤n,q≤105,0≤ai​≤230−1 。

思路

首先特判 l=rl = rl=r 。
假设分成的两个集合 S,TS, TS,T ， SSS 中元素的或等于 TTT 中元素的与，最后得到的结果为 valvalval ，记 popcount(val)popcount(val)popcount(val) 表示 valvalval 二进制表示下 111 的个数 kkk ，假设 popcount(val)=kpopcount(val) = kpopcount(val)=k ，容易发现 SSS 集合内所有数的 popcountpopcountpopcount 都需要小于等于 kkk ， TTT 集合内所有数的 popcountpopcountpopcount 都需要大于等于 kkk 。
于是可以考虑枚举 kkk ，分三类讨论：

  • 区间内 popcount=kpopcount = kpopcount=k 的数个数为 000 ，直接判断两边的结果是否相同。
  • 区间内 popcount=kpopcount = kpopcount=k 的数个数为 111 ，枚举这个数放在哪个个集合中。
  • 区间内 popcount=kpopcount = kpopcount=k 的数个数为 222 ，首先判断是否所有 popcount=kpopcount = kpopcount=k 的数都相等，若不相等则无论怎么放都不能满足要求，可以反证，再枚举这些数放在哪个集合中或者都放。

于是可以 313131 棵线段树维护 popcount=kpopcount = kpopcount=k 的所有数的或 / 与值和个数。
复杂度 O(n⋅log2n)O(n \cdot log^2n)O(n⋅log2n) 。

I. Modulo Permutations

tag

  • 思维
  • dp

题意

计算满足如下条件的长为 nnn 的排列个数，模 109+710^9 + 7109+7 ：

  • ∀i∈[1,n],pi mod pi+1≤2,(pn+1=p1)\forall i \in [1, n], p_i \ mod \ p_{i+1} \leq 2, (p_{n + 1} = p_1)∀i∈[1,n],pi​ mod pi+1​≤2,(pn+1​=p1​) 。

其中 1≤n≤1061 \leq n \leq 10^61≤n≤106 。

思路

若 nnn 小于等于 333 ，则可以随便排，若 nnn 大于 333 ，则所有大于等于 333 的数后面只能放比它小的数字，所以最后的结果一定可以表示成两段递减的序列拼接起来的结果，且两段的结尾一定是 111 或 222 ，即 (...,1)+(...,2)(..., 1) + (..., 2)(...,1)+(...,2) 。
于是可以考虑 dpx,y(x<y)dp_{x, y}(x < y)dpx,y​(x<y) 表示两段结尾值分别为 x,yx, yx,y 的分配方案数，有两种转移：

  • dp[x][y]→dp[x−1][y]dp[x][y] \rightarrow dp[x - 1][y]dp[x][y]→dp[x−1][y] ， x mod x−1x \ mod \ x - 1x mod x−1 一定满足条件，这一步相当于数组整体平移一个单位。
  • dp[x][y]→dp[x][x−1]dp[x][y] \rightarrow dp[x][x - 1]dp[x][y]→dp[x][x−1] ， y mod x−1y \ mod \ x - 1y mod x−1 需要小于等于 222 ，可以枚举 x−1x - 1x−1 的倍数进行转移，复杂度为调和级数。

第一步转移一定满足条件且相当于数组整体平移一个单位，因此可以省略，记 dpx,x−1dp_{x, x - 1}dpx,x−1​ 为两段结尾值分别为 x,x−1x, x - 1x,x−1 的分配方案数，转移时枚举 x−1x - 1x−1 的倍数即可。
复杂度 O(n⋅logn)O(n \cdot logn)O(n⋅logn) 。

L. Neo-Robin Hood

tag

  • 贪心
  • 二分

题意

总共 nnn 个二元组 (mi,pi)(m_i, p_i)(mi​,pi​) ，选择恰好 2⋅k2 \cdot k2⋅k 个二元组使得其中 kkk 个的 ∑m\sum m∑m 大于等于另 kkk 个的 ∑p\sum p∑p 。
其中 1≤n≤105,1≤mi,pi≤1091 \leq n \leq 10^5, 1 \leq m_i, p_i \leq 10^91≤n≤105,1≤mi​,pi​≤109 。

思路

考虑同时选了两个人 i,ji, ji,j 作为匹配，则他们的贡献为 mi−pjm_i - p_jmi​−pj​ 或 mj−pim_j - p_imj​−pi​ ，若 mi−pj≥mj−pim_i - p_j \geq m_j - p_imi​−pj​≥mj​−pi​ 则移项得 mi+pi≥mj+pjm_i + p_i \geq m_j + p_jmi​+pi​≥mj​+pj​ ，可以得到总是 m+pm + pm+p 大的人取 mmm 值，另一个人取 ppp 值。
于是可以先对 m+pm + pm+p 降序排序，所取的人一定来自于前后两段不相交的区间中，二分取的个数 kkk ，堆维护一个前缀 / 后缀的 kkk 大值 / 小值的和即可。
复杂度 O(n⋅log2n)O(n \cdot log^2n)O(n⋅log2n) 。

tag

  • 贪心
  • 模拟

题意

给定一张 nnn 个 mmm 条边的 dagdagdag ，给出在图上跑 kkk 次拓扑序的结果（多次的结果合并，但各个结果内部的顺序不变），求出每个元素属于第几次结果，多解输出任意一种。
其中 1≤n⋅k≤5⋅105,0≤m≤2500001 \leq n \cdot k \leq 5 \cdot 10^5, 0 \leq m \leq 2500001≤n⋅k≤5⋅105,0≤m≤250000 。

思路

结果序列从首到尾贪心地匹配即可，每个编号的节点维护一个当前它可以属于的结果的集合。
