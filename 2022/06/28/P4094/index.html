题解 洛谷P4094 [HEOI2016/TJOI2016]字符串

发表于 2022-06-28 更新于 2024-07-21 分类于 杂题 本文字数： 5k 阅读时长 ≈ 5 分钟

后缀数组 + 主席树上二分

题意

已知小写字符串 sss 。
mmm 次询问，每次询问串 sss 中子串 s[c...d]s[c...d]s[c...d] 与子串 s[a...b]s[a...b]s[a...b] 中每个子串的 lcplcplcp 长度的最大值。
其中 1≤∣s∣,m≤1051 \leq |s|, m \leq 10^51≤∣s∣,m≤105 。

思路

由于字符串 sss 是给定的，可以 O(nlogn)O(nlogn)O(nlogn) 地预处理出 sss 的后缀数组，从而可以 O(1)O(1)O(1) 地求出某两个后缀的 lcplcplcp 长度。
很容易发现答案具有二分性，可以二分出 lcplcplcp 的长度从而转换为存在性问题。
假设当前二分的结果是 midmidmid ，可以发现 只有起点位于 [a...b−mid+1][a...b-mid+1][a...b−mid+1] 内的子串与 s[c...d]s[c...d]s[c...d] 的 lcplcplcp 长度可能大于等于 midmidmid ，所以问题即转换为求 [a...b−mid+1][a...b-mid+1][a...b−mid+1] 内的后缀与后缀 ccc 的 lcplcplcp 长度。
因为是多组询问，不可能每一次都将 ccc 与 [a...b−mid+1][a...b-mid+1][a...b−mid+1] 中的每一个后缀比较一次。考虑后缀数组的性质， rka,rka+1,...rkb−mid+1rk_a, rk_{a+1}, ...rk_{b-mid+1}rka​,rka+1​,...rkb−mid+1​ 与 rkcrk_crkc​ 放置在数轴上是一系列散点，由 lcp(rki,rkj)==min(heightk)(rki≤k≤rkj)lcp(rk_i,rk_j) == min(height_k) (rk_i \leq k \leq rk_j)lcp(rki​,rkj​)==min(heightk​)(rki​≤k≤rkj​) 可以发现只需要计算 rkcrk_crkc​ 左右最近的两个位置与它的 lcplcplcp 即可。具体实现时，可以对串 sss 的下标可持久化建树，每次加入 rkirk_irki​ ，单次询问需要查询 xxx 左右的第一个 111 的位置，查询一次复杂度为 O(logn)O(logn)O(logn) 。所以处理一次询问的复杂度为 O(log2n)O(log^2n)O(log2n) 。
预处理加 nnn 次询问的总复杂度为 O(n⋅log2n)O(n \cdot log^2n)O(n⋅log2n) 。
